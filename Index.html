Day 01 What is ML? Bias-Variance & the ML Workflow

📝 Key Notes

ML is the art of learning patterns from data instead of hard-coding rules. The Bias-Variance tradeoff is the most important concept in all of ML — high bias underfits (too simple), high variance overfits (memorises noise). The goal is always the sweet spot.

Supervised = labelled data; Unsupervised = find structure without labels
ML workflow: collect → clean → split (train/val/test) → train → evaluate → iterate
Never touch the test set until the very end — it's your reality check
Bias-Variance: more model complexity = lower bias, higher variance
Day 02 Linear Regression & Gradient Descent
visualconcept
📺 Watch
Linear Regression, Clearly Explained
StatQuest
~27 min
Gradient Descent, Step by Step
StatQuest
~20 min
📝 Key Notes

Linear regression fits a line by minimising the sum of squared residuals. Gradient descent is how we do that numerically — slide down the loss surface by computing the slope and stepping in the opposite direction. The learning rate controls step size.

Loss function (MSE) = average squared difference between predictions and actual
Gradient descent: θ = θ − α·∇L, where α is the learning rate
Too large learning rate → overshoots; too small → slow convergence
Stochastic GD uses one sample at a time; mini-batch uses small batches (faster)

ROC and AUC, Clearly Explained

📝 Key Notes

Logistic regression predicts probabilities using the sigmoid function, squashing any real number into [0,1]. Despite the name, it's a classification algorithm. For evaluation, accuracy alone is misleading — use precision/recall for imbalanced datasets, ROC-AUC for overall performance.

Sigmoid: σ(z) = 1 / (1 + e^−z) → outputs probability
Confusion matrix: TP, FP, TN, FN — memorise this fully
Precision = TP / (TP+FP); Recall = TP / (TP+FN)
AUC = 1 is perfect; AUC = 0.5 = random guessing

📝 Key Notes

Decision trees split data by asking questions. Each split is chosen to maximise information gain (reduce Gini impurity or entropy). Trees are highly interpretable but overfit easily — solved by limiting depth. Cross-validation gives a more reliable estimate of generalisation than a single validation split.

Gini impurity = 1 − Σ(p²) — measures how mixed a node is
Deep trees = overfitting; prune with max_depth, min_samples_leaf
k-Fold CV: split data into k parts, train on k-1, test on 1, rotate
Use CV to select hyperparameters, not to test final model

Random Forests = many trees trained on bootstrapped samples with random feature subsets. By averaging many uncorrelated trees, variance drops dramatically. Bagging reduces variance; Boosting (AdaBoost, XGBoost) reduces bias by sequentially correcting mistakes.

Bootstrap = sampling with replacement → creates different training sets per tree
Out-of-bag error ≈ free cross-validation estimate built into Random Forests

📝 Key Notes

A neural network is just a massive composition of matrix multiplications + non-linearities. Each neuron computes a weighted sum and passes it through an activation function. The network's job during training is to find weights that minimise the loss.

Layers: input → hidden(s) → output; depth = power to learn complex patterns
Activation functions (ReLU, sigmoid, tanh) allow the network to learn non-linear patterns
ReLU (max(0,x)) is default for hidden layers — fast, no vanishing gradient
Universal approximation theorem: a wide enough NN can approximate any function
Day 09 Backpropagation: How Neural Nets Learn
visualconcept
📺 Watch
Backpropagation — Chapter 3
3Blue1Brown
~14 min
Backpropagation Calculus — Chapter 4
3Blue1Brown
~10 min
📝 Key Notes

Backpropagation is the chain rule of calculus applied to a computational graph. We push the error backwards from output to input, computing how much each weight contributed to the loss. This gives us gradients → update weights with gradient descent.

Forward pass: compute predictions; Backward pass: compute all gradients
Chain rule: ∂L/∂w = (∂L/∂y) × (∂y/∂w) — gradients multiply along the path
Vanishing gradient: deep sigmoid networks struggle because gradients shrink layer by layer
PyTorch autograd handles all this automatically via computational graphs

Day 11 CNNs — Convolutional Neural Networks
visualconcept
📺 Watch
Convolutional Neural Networks (CNNs) Explained
3Blue1Brown
~24 min
📝 Key Notes

CNNs exploit spatial structure in images. A filter/kernel slides over the image, computing dot products — this detects features (edges, textures, shapes) regardless of where they appear. Pooling reduces spatial dimensions. Deep CNNs build a hierarchy: edges → shapes → objects.

Parameter sharing: one filter scans the whole image = massive parameter reduction
Feature maps: output of a conv layer, each map = one filter's response to the image
MaxPooling = take the max in each region → translation invariance
BatchNorm normalises activations between layers → stable, faster training

📝 Key Notes

Modern training relies on smarter optimisers than vanilla SGD. Adam combines momentum (past gradients) with adaptive learning rates per parameter. Dropout randomly zeros neurons during training — forces redundancy and prevents co-adaptation.

Momentum: accumulate velocity in gradient direction → faster convergence
Adam = adaptive moment estimation; works well as default out-of-the-box
Dropout rate of 0.2–0.5 at training time; at test time all neurons are on
Learning rate schedule: decay over time or use OneCycleLR for faster training

📝 Key Notes

Instead of training from scratch, take a model pre-trained on millions of images (e.g. ResNet) and fine-tune its last layers on your small dataset. This is how you get state-of-the-art results with very little data. The key insight: early layers learn universal features (edges, textures).

Day 14 RNNs & LSTMs — Sequence Models Intro
visual

📝 Key Notes

RNNs process sequences by maintaining a hidden state that gets updated at each step. The problem: gradients vanish over long sequences. LSTMs solve this with gates (forget, input, output) that selectively remember and forget — allowing learning of long-range dependencies.

RNN hidden state: h_t = tanh(W·h_{t-1} + U·x_t)
LSTM cell state = memory highway; gates control what flows through
GRU = simplified LSTM with fewer parameters, similar performance

Day 15 Word Embeddings: Word2Vec & Semantic Space
visual
📺 Watch
Word Embedding & Word2Vec — StatQuest NLP Series
StatQuest
~22 min
📝 Key Notes

Words can be represented as dense vectors in a high-dimensional space where semantically similar words are close together. Word2Vec learns these by predicting context (CBOW) or centre words (Skip-gram). Famous property: King − Man + Woman ≈ Queen.

One-hot encoding → huge sparse vectors; embeddings → compact dense vectors
Cosine similarity measures angle between vectors = semantic similarity
Embedding dimension is a hyperparameter — 50 to 300 for word2vec, 768 for BERT
Modern LLMs use contextual embeddings — same word gets different vectors in different contexts
Day 16 Attention Mechanism — The Key Innovation
visualLLM
📺 Watch
Attention in Transformers, Step by Step — Chapter 6
3Blue1Brown
~27 min
📝 Key Notes

Attention lets each token look at every other token and decide how relevant they are. It computes Queries, Keys, and Values — Q·Kᵀ gives a score (how relevant), softmax normalises it, and we take a weighted sum of Values. This is how "The bank by the river" knows which meaning of "bank" to use.

Attention(Q,K,V) = softmax(QKᵀ / √d_k) · V
Scaling by √d_k prevents the dot products from growing too large → softmax saturation
Multi-head attention = run attention multiple times in parallel with different projections
Causal masking in GPT: each token can only attend to past tokens (autoregressive)
Day 17 Transformer Architecture: The Full Picture
visualLLM
📺 Watch
Transformers, the Tech Behind LLMs — Chapter 5
3Blue1Brown
~27 min
📝 Key Notes

A Transformer block = Multi-head self-attention + Feed-forward network + Residual connections + LayerNorm. Stack 12–96 of these blocks and you get GPT. The residual stream carries information throughout; each block updates it. Positional encoding adds order information since attention is permutation-invariant.

Residual connections (x + F(x)) allow gradients to flow directly → train very deep nets
LayerNorm normalises per-token (vs BatchNorm which normalises per-feature)
FFN in each block = two linear layers with ReLU/GELU; adds non-linearity
Context window = max number of tokens the model can "see" at once

Byte Pair Encoding merges the most frequent pairs of bytes/characters iteratively. GPT-4's tokeniser has 100k merges. Weird LLM behaviours (can't count letters, odd spelling) trace back to tokenisation artefacts. Understanding this explains a surprising number of LLM quirks.

Day 23 RLHF — How ChatGPT Learned to Follow Instructions
LLMconcept
📺 Watch
RLHF Explained — Reinforcement Learning from Human Feedback
Hugging Face
~20 min
📝 Key Notes

RLHF is the secret sauce behind ChatGPT. Step 1: SFT on demonstrations. Step 2: train a Reward Model on human preference data (which answer is better?). Step 3: use PPO to optimise the LLM against the reward model. This aligns the model with human values.

SFT = Supervised Fine-Tuning: teach model to follow instruction format
Reward Model: trained on human A/B preference data; outputs a scalar score
PPO (Proximal Policy Optimisation) updates the LLM to maximise reward without drifting too far
DPO (Direct Preference Optimisation) = newer, simpler alternative to RLHF
Day 24 RAG — Retrieval-Augmented Generation

Rabbitmetrics

📝 Key Notes

LLMs hallucinate because they rely only on learned weights. RAG fixes this: embed your documents into a vector database, retrieve the most relevant chunks at query time, and pass them as context to the LLM. It's now the dominant pattern for production AI systems.

Chunk documents → embed each chunk → store in vector DB (Chroma, Pinecone, Weaviate)
At query time: embed query → ANN search → retrieve top-k chunks → stuff into prompt
Cosine similarity between query embedding and chunk embeddings = relevance score
Advanced RAG: reranking, HyDE (hypothetical document embeddings), parent-child chunks

Chain-of-Thought prompting tells the model to "think step by step" — dramatically improves multi-step reasoning. Few-shot examples in the prompt teach the model the exact format you want. In-context learning is when the model adapts to examples without any weight updates.

Zero-shot: no examples; Few-shot: 1–5 input-output examples in the prompt
Chain-of-Thought: add "Let's think step by step" → big gains on math/logic tasks
System prompt sets model's persona and constraints; user prompt is the actual task
Temperature=0 for deterministic/factual; Temperature>0.7 for creative outputs
Day 26 Fine-tuning LLMs: LoRA, QLoRA & PEFT
LLMcode
📺 Watch
Fine-tuning LLMs with LoRA Explained
Weights & Biases
~30 min
📝 Key Notes

LoRA (Low-Rank Adaptation) fine-tunes only tiny adapter matrices inserted into the model — just 0.1% of parameters — instead of all 7B+. The original weights stay frozen. QLoRA adds 4-bit quantisation so you can fine-tune a 7B model on a single GPU. This is how the open-source community trains Llama models.

LoRA: add ΔW = BA (low-rank decomposition) to each weight matrix; only train B,A
Rank r=8 or 16 is usually enough; higher rank = more expressiveness but more params
QLoRA: quantise base model to 4-bit, then apply LoRA in 16-bit = 4× memory reduction
Use HuggingFace PEFT + trl libraries; 50–100 examples often enough for task-specific fine-tuning
